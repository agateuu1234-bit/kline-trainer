## Design Decisions（plan-time 锁定，codex review 抓变动）

### §1 解压库选 ZipFoundation 0.9.x
- 来源：weichsel/ZIPFoundation，MIT，pure Swift，SwiftPM 一行
- 备选 reject 理由：
  - `Process` + `/usr/bin/unzip`：iOS 不可用
  - Apple AppleArchive：仅支持 .aar/.aau，不支持标准 zip
  - Apple Compression API：原始流，无 zip 容器解析
  - 自写 zip 解析：scope creep，spec 没要求 minimal zip 格式
- 版本 pin：`0.9.0..<1.0.0`（与 GRDB 同级 upToNextMajor 节奏）

### §2 CRC32 算法（R3 修订：streaming 64KB chunk 防 OOM）
- spec L739 字面：`format(zlib.crc32(zip_bytes) & 0xFFFFFFFF, '08x')` —— IEEE polynomial、对**整个 zip 文件字节流**算（不是 zip entry header 内嵌的 CRC）
- 实现：`Data.crc32(checksum: prev) -> CRC32` —— ZipFoundation 提供的 IEEE polynomial Data extension，等价 zlib.crc32；**checksum 参数是 previous chunk 的 CRC，可累加**
- **streaming**（R3 codex finding 3 推动）：`FileHandle(forReadingFrom:)` 64KB chunk 循环读，每 chunk 累加 CRC32；避免 `Data(contentsOf:)` 一次读全文件进 RAM 撞 OOM
- 输出：`String(format: "%08x", checksum)` 强制 8 字符小写
- 比对：lowercase 后 `==`（大小写敏感会 false-positive 拒掉合法 hash）

### §3 verifyNonEmpty 阈值定义（R1 修订：按 spec literal 强 invariant）

**Spec literal 锚点**（R1 codex finding 1 推动我去查的真锚）：
- modules_v1.4.md §B2 L741："**不变量：月线前 ≥30 / "之后"= 8 根月 K 时间窗口**"
- plan_v1.5.md L1035："起始月K线之前**至少 30 根月K线**，之后**至少 8 根月K线**"
- plan_v1.5.md L1062：`assert before_count >= 30, f"Not enough {period} bars before start"`（B2 backend 对**所有周期**都 assert ≥30 before）

**P2 P3b 客户端 enforce（defense-in-depth；spec L741 是 B2 invariant，P2 拒收 B2 bug 漏过 / 传输损坏的 trash）：**

对每个 `period in Period.allCases`：
1. `candles[period]` 必须有 key 且数组非空 → 否则 `.trainingSet(.emptyData)`
2. **`startDatetime` 前 ≥30 candles**（per spec L1062 字面，所有周期）→ 否则 `.emptyData`
3. **monthly 周期**：`startDatetime` 起（含）后 ≥8 candles（per spec L741 字面）→ 否则 `.emptyData`
4. **其它 5 周期**：`startDatetime` 起（含）后 ≥1 candle（spec 未明确具体数字，"时间窗口"字面取最小语义防 0-after 的 trivial trash）→ 否则 `.emptyData`

**算法**：用 `KLineCandle.datetime: Int64` 与 `TrainingSetMeta.startDatetime: Int64` 比较；before = `datetime < startDatetime`；after = `datetime >= startDatetime`。

**为什么不是简单 `!arr.isEmpty`（R0 旧设计）**：单 candle sqlite 能 trivially pass，但下游训练 (E5 / E6) 依赖 startDatetime 前 30 根才能算 ma66 / boll / macd 指标，单 row 直接 NaN。R1 codex finding 1 抓的就是这个。

**Period.allCases = [.m3, .m15, .m60, .daily, .weekly, .monthly]（6 周期）**

**测试 fixture 需求**：FakeReader 必须能注入 ≥30 before + ≥8 (monthly) / ≥1 (其它) after 的 candles。Test helper `makeValidCandles(period:startDatetime:)` 生成 30 before + N after 的合法序列。

### §4 ZipExtractor 输出选择策略（R1 修订：strict shape）

**Spec 锚点**：modules L716 "B2 压缩 zip" + plan L1087 "压缩 → 生成 100 个训练组"——B2 zip 形态 = exactly 1 top-level sqlite file。

**R0 旧设计的攻击面**（R1 codex finding 2 抓的）：
- 多 sqlite 时取字典序第一 → 攻击者注入 `aaa.sqlite + 真.sqlite` 让 `aaa.sqlite` 被 P2 验证 + cache + confirm
- 嵌套 sqlite 在 subdir → top-level scan 错过 → `.unzipFailed` 错杀合法 zip（如果 B2 将来真用 subdir）

**R1 修订决策（strict shape，spec 字面 enforce）**：
- 临时目录 = `FileManager.default.temporaryDirectory.appendingPathComponent("ZipExtract-\(UUID().uuidString)")`，调用方负责后续 cleanup
- 解压后**只递归扫描 regular files**（用 `.isRegularFileKey` 过滤，跳过 dir / symlink）
- 找出**所有** path extension == `.sqlite`（lowercased）的 regular file
- **必须 exactly 1** → 返回该 URL
- 0 个 → `.unzipFailed`（zip 内无 sqlite）
- ≥2 个 → `.unzipFailed`（B2 形态保证仅 1，多个属攻击或损坏）
- dir 名为 `xxx.sqlite` → 不计（仅 regular file 算）
- 嵌套 sqlite（subdir 内）→ 计入计数，与 top-level 平等（B2 形态 invariant 是"exactly 1"，路径无关）

**为什么递归而不是 top-level only**：codex 提"非递归会错杀合法嵌套 zip"——但 B2 spec 没承诺 top-level/subdir。递归 + 强 count 是两个攻击路径都关：多 sqlite 不允许 + 子目录沉默 hijack 不允许。

### §5 cleanup 错误处理（R3+R4 修订：path containment + strict descendant guard）
- spec L1771-1774 字面："非致命：失败只打日志，不抛"
- 实现：for-each FileManager.removeItem，catch 走 `os_log` `.error` 级别（不抛、不返 bool）
- 重要：`removeItem` 对不存在路径会抛 `NSFileNoSuchFileError`，统一归 log 不抛
- **path containment guard**（R3 codex finding 1 推动）：URL 必须 in `NSTemporaryDirectory()` 子树才删；外部路径 → log "rejected" + skip。理由：runner / recovery / journal bug 可能传 cache / DB / Documents 路径致数据丢失；使用 `temporaryDirectory` 而非 `ZipExtract-*` 前缀以兼容 P1 download zip 路径（P1 未交付，命名前缀未冻结）
- **strict descendant guard**（R4 codex finding 2 推动）：必须 `hasPrefix(tempRoot + "/")` 且 `!= tempRoot`；==tempRoot 本身一律拒删（否则 caller bug 传 tempRoot URL 会清空整个临时目录，影响其它 PR / 系统组件 / iOS sandbox 缓存）
- 路径比较用 `resolvingSymlinksInPath().standardizedFileURL.path` 防 `/tmp` symlink + path traversal

### §6 边界错误翻译（M0.4 gate stub 闭合）
- 参考 `Internal/PersistenceErrorMapping.swift` 既有模式
- ZipFoundation 暴露 `Archive.ArchiveError` enum；Foundation 文件 IO 抛 `NSError(domain: NSCocoaErrorDomain, ...)`
- 映射表：
  - `Archive.ArchiveError.unreadableArchive / .invalidArchive` → `.trainingSet(.unzipFailed)`
  - `NSCocoaErrorDomain` `.fileReadNoSuchFileError` / `.fileNoSuchFileError` → `.trainingSet(.fileNotFound)`
  - 其它 IO（`NSFileWriteOutOfSpaceError`）→ `.persistence(.diskFull)`
  - 兜底：`.persistence(.ioError("zip_io_error"))`（脱敏，不暴露原始 message）
- ZipFoundation Error 类型不跨 module 暴露（KlineTrainerPersistence internal，contracts 消费者只接 AppError）

### §7 测试 fixture 策略
- `ZipFixture.swift` 工具方法：
  - `makeMinimalSqliteZip(at directoryURL: URL) -> (zipURL: URL, expectedCRC32Hex: String)` —— 用 ZipFoundation 写一个含 1 个 sqlite 文件的 zip（sqlite 内容：调既有 `TrainingSetSQLiteFixture` 生成，或最简单 SQLite header 字节流）
  - `crc32Hex(of fileURL: URL) -> String` —— 算 IEEE CRC32 8-char lowercase
- 不复用 `TrainingSetSQLiteFixture`：避免 P2/P3 fixture 耦合；ZipFixture 内自管 sqlite header

```swift
import Foundation

public protocol ZipIntegrityVerifying: Sendable {
    /// 校验 zipURL 的 CRC32 是否等于 expectedCRC32Hex（8 字符小写 IEEE polynomial）。
    /// 算法：对整个 zip 文件字节流算 zlib.crc32 等价 IEEE，输出 `%08x` 小写。
    /// - throws AppError.trainingSet(.crcFailed) 不匹配
    /// - throws AppError.trainingSet(.fileNotFound) zip 文件不存在
    /// - throws AppError.persistence(.ioError) 其它读取失败
    func verify(zipURL: URL, expectedCRC32Hex: String) throws
}
```

`ZipExtracting.swift`:
```swift
// Kline Trainer Swift Contracts — P2 port 2
// Spec: kline_trainer_modules_v1.4.md §P2 line 1759-1763
//       B2 形态契约 modules L716 + plan_v1.5 L1087："压缩 zip" = exactly 1 sqlite

import Foundation

public protocol ZipExtracting: Sendable {
    /// 解压 zipURL 到新建的临时目录，返回唯一 sqlite 文件 URL。
    ///
    /// **strict shape 契约**（spec L716 B2 zip 形态 enforce）：
    /// - zip 必须 exactly 1 个 regular file 且 path 后缀 `.sqlite`（lowercased）
    /// - 0 个 sqlite / ≥2 个 sqlite / 含任何其它 regular file → throw `.unzipFailed`
    /// - dir entry / symlink entry 不计；嵌套路径（subdir 内）也算 entry
    /// - **必须解压前校验 archive 形态**，避免恶意 zip 用大量 junk sidecar 消耗临时磁盘
    ///
    /// 临时目录归调用方管理（典型经 DownloadAcceptanceCleaning.cleanup 释放）。
    ///
    /// - throws AppError.trainingSet(.unzipFailed) shape 不符 / 解压失败 / archive 损坏
    /// - throws AppError.trainingSet(.fileNotFound) zip 文件不存在
    /// - throws AppError.persistence(.diskFull) 临时目录写满
    /// - throws AppError.persistence(.ioError) 其它 IO 失败
    func extract(zipURL: URL) throws -> URL
}
```

```swift
import Foundation
import ZIPFoundation
import KlineTrainerContracts

/// ZipFoundation / Foundation IO Error → AppError 边界翻译
/// （per docs/governance/m04-apperror-translation-gate.md，参考 PersistenceErrorMapping 模式）。
/// 仅 KlineTrainerPersistence 内部使用；不暴露 ZipFoundation 类型给 contracts 消费者。
enum ZipErrorMapping {
    static func translate(_ error: Error, fileURL: URL? = nil) -> AppError {
        if let app = error as? AppError {
            return app                                  // 已是 AppError 直通（防双翻译）
        }
        if let archiveErr = error as? Archive.ArchiveError {
            switch archiveErr {
            case .unreadableArchive, .invalidEntryPath:
                return .trainingSet(.unzipFailed)
            default:
                return .trainingSet(.unzipFailed)       // 其它 Archive 错误统一 unzipFailed
            }
        }
        let nsErr = error as NSError
        if nsErr.domain == NSCocoaErrorDomain {
            switch nsErr.code {
            case NSFileReadNoSuchFileError, NSFileNoSuchFileError:
                return .trainingSet(.fileNotFound)
            case NSFileWriteOutOfSpaceError:
                return .persistence(.diskFull)
            default:
                break
            }
        }
        return .persistence(.ioError("zip_io_error"))   // 脱敏，不放原始 message
    }
}
```

```swift
// ios/Contracts/Sources/KlineTrainerPersistence/DefaultZipIntegrityVerifier.swift
import Foundation
import ZIPFoundation
import KlineTrainerContracts

public struct DefaultZipIntegrityVerifier: ZipIntegrityVerifying {
    private static let chunkSize = 64 * 1024

    public init() { }

    public func verify(zipURL: URL, expectedCRC32Hex: String) throws {
        let crc: CRC32
        do {
            crc = try Self.streamingCRC32(of: zipURL)
        } catch {
            throw ZipErrorMapping.translate(error, fileURL: zipURL)
        }
        let actualHex = String(format: "%08x", crc)
        let expectedNormalized = expectedCRC32Hex.lowercased()
        guard actualHex == expectedNormalized else {
            throw AppError.trainingSet(.crcFailed)
        }
    }

    private static func streamingCRC32(of url: URL) throws -> CRC32 {
        // R4 codex finding 3：不显式 throw NSError 以兼容 Task 6 grep gate；
        // FileHandle(forReadingFrom:) 在文件不存在 / 不可读时会自然抛 NSCocoaErrorDomain
        // (NSFileNoSuchFileError / NSFileReadNoPermissionError)，由外层 verify 的
        // ZipErrorMapping.translate 翻译成 AppError.trainingSet(.fileNotFound) / .ioError
        let handle = try FileHandle(forReadingFrom: url)
        defer { try? handle.close() }

        var crc: CRC32 = 0
        while true {
            let chunk = handle.readData(ofLength: chunkSize)
            if chunk.isEmpty { break }
            crc = chunk.crc32(checksum: crc)
        }
        return crc
    }
}
```

```swift
// ios/Contracts/Sources/KlineTrainerPersistence/DefaultZipExtractor.swift
import Foundation
import ZIPFoundation
import KlineTrainerContracts

public struct DefaultZipExtractor: ZipExtracting {
    public init() { }

    public func extract(zipURL: URL) throws -> URL {
        guard FileManager.default.fileExists(atPath: zipURL.path) else {
            throw AppError.trainingSet(.fileNotFound)
        }

        // R2: 解压前 inspect Archive entries — 严格 shape (exactly 1 regular .sqlite)，
        // 不允许任何其它 regular file payload（防 junk sidecar 消耗临时磁盘 / 混淆审计）。
        guard let archive = Archive(url: zipURL, accessMode: .read) else {
            throw AppError.trainingSet(.unzipFailed)
        }

        var sqliteEntries: [Entry] = []
        var hasOtherRegularFile = false
        for entry in archive {
            guard entry.type == .file else { continue }     // dir/symlink 不计
            if entry.path.lowercased().hasSuffix(".sqlite") {
                sqliteEntries.append(entry)
            } else {
                hasOtherRegularFile = true
                break
            }
        }
        guard sqliteEntries.count == 1, !hasOtherRegularFile else {
            throw AppError.trainingSet(.unzipFailed)
        }
        let sqliteEntry = sqliteEntries[0]

        // 形态校验通过，单 entry 解压。
        let outputDir = FileManager.default.temporaryDirectory
            .appendingPathComponent("ZipExtract-\(UUID().uuidString)")
        do {
            try FileManager.default.createDirectory(
                at: outputDir, withIntermediateDirectories: true
            )
            // entry.path 可能含 subdir（如 "sub/x.sqlite"）；只取 lastPathComponent 平展输出
            let lastName = (sqliteEntry.path as NSString).lastPathComponent
            let outputURL = outputDir.appendingPathComponent(lastName)
            _ = try archive.extract(sqliteEntry, to: outputURL)
            return outputURL
        } catch {
            try? FileManager.default.removeItem(at: outputDir)
            throw ZipErrorMapping.translate(error, fileURL: zipURL)
        }
    }
}
```

**实现要点**：
- `Archive(url:accessMode:.read)` 仅打开 zip 不解压 entry 内容
- 遍历 entries 用 `entry.type == .file` 过滤，dir/symlink 不计
- `entry.path.lowercased().hasSuffix(".sqlite")` 判 sqlite（注：`.sqlitejournal`不会误匹配）
- 任一非 sqlite 的 regular file → 立刻 break + reject（pre-extraction，无磁盘消耗）
- exactly 1 sqlite + 0 其它 regular file → 仅解压该 entry 到 outputDir
- entry.path 含 subdir 时取 lastPathComponent 平展输出（避免恶意 path traversal 风险，ZipFoundation 自身也 sanitize 但 belt-and-suspenders）

```swift
// ios/Contracts/Sources/KlineTrainerPersistence/DefaultTrainingSetDataVerifier.swift
import Foundation
import KlineTrainerContracts

public struct DefaultTrainingSetDataVerifier: TrainingSetDataVerifying {
    public init() { }

    public func verifyNonEmpty(reader: TrainingSetReader) throws {
        let meta: TrainingSetMeta
        do {
            meta = try reader.loadMeta()
        } catch let app as AppError {
            throw app
        } catch {
            throw AppError.persistence(.dbCorrupted)
        }

        let candles: [Period: [KLineCandle]]
        do {
            candles = try reader.loadAllCandles()
        } catch let app as AppError {
            throw app
        } catch {
            throw AppError.persistence(.dbCorrupted)
        }

        let startDT = meta.startDatetime
        for period in Period.allCases {
            guard let arr = candles[period], !arr.isEmpty else {
                throw AppError.trainingSet(.emptyData)
            }
            // spec L1062：每周期 startDatetime 前 ≥30 candles
            let beforeCount = arr.lazy.filter { $0.datetime < startDT }.count
            guard beforeCount >= 30 else {
                throw AppError.trainingSet(.emptyData)
            }
            // spec L741：monthly 之后 ≥8；其它周期 ≥1（spec spirit 防 0-after trash）
            let afterCount = arr.lazy.filter { $0.datetime >= startDT }.count
            let requiredAfter = (period == .monthly) ? 8 : 1
            guard afterCount >= requiredAfter else {
                throw AppError.trainingSet(.emptyData)
            }
        }
    }
}
```

```swift
// ios/Contracts/Sources/KlineTrainerPersistence/DefaultDownloadAcceptanceCleaner.swift
import Foundation
import os.log
import KlineTrainerContracts

public struct DefaultDownloadAcceptanceCleaner: DownloadAcceptanceCleaning {
    private static let log = OSLog(
        subsystem: "com.klinetrainer.persistence",
        category: "DownloadAcceptanceCleaner"
    )

    private let tempRootPath: String

    public init() {
        // R3：path containment guard 用 NSTemporaryDirectory 子树作 owned root
        // resolvingSymlinksInPath 防 /tmp → /private/tmp 这类 macOS symlink 绕过
        self.tempRootPath = FileManager.default.temporaryDirectory
            .resolvingSymlinksInPath()
            .standardizedFileURL
            .path
    }

    public func cleanup(tempURLs: [URL]) {
        let fm = FileManager.default
        for url in tempURLs {
            let resolved = url.resolvingSymlinksInPath()
                .standardizedFileURL.path
            // R3 codex finding 1 + R4 codex finding 2：strict descendant，不允许 == tempRoot 本身
            // （否则 caller bug 传 tempRoot URL 会清空整个临时目录，影响其它 PR/系统组件）
            guard resolved.hasPrefix(tempRootPath + "/"), resolved != tempRootPath else {
                os_log(
                    "cleanup rejected non-temp-descendant path: %{public}@",
                    log: Self.log, type: .error, resolved
                )
                continue
            }
            do {
                try fm.removeItem(at: url)
            } catch {
                let nsErr = error as NSError
                if nsErr.domain == NSCocoaErrorDomain &&
                   (nsErr.code == NSFileNoSuchFileError ||
                    nsErr.code == NSFileReadNoSuchFileError) {
                    continue                           // 不存在路径不视为错误
                }
                os_log(
                    "cleanup failed: %{public}@ code=%{public}d",
                    log: Self.log, type: .error,
                    String(describing: url.lastPathComponent), nsErr.code
                )
            }
        }
    }
}
```
