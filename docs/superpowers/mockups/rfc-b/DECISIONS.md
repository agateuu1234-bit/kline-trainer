## 决策清单

### D1 · 坐标轴：透明文字、无底框、纵轴移左 —— ✓LOCKED
保持现有坐标轴**位置/布局**基础上：
1. 价/时间/周期标签的**不透明小底框去掉**，改成悬浮在 K 线上的**透明文字**（同花顺式）→ 不遮 K 线、面积更大。
2. **纵坐标（价轴）从右侧移到左侧**——因为最新一根 K 线在最右边，价轴放右会挡最新 K 线；移左即不挡。时间轴仍在底部。
   - 周期标记（如「60分」）原在左上，与左侧价标可能打架 → 移到**右上角**避让。
- 影响面（实现时核实）：`AxisGridLayout`（价标 x 坐标改到左缘 + 去背景）、`KLineView+Crosshair.swift` 的 `drawLabelBox` 背景填充、`KLineView+AxisGrid.swift` 标签绘制。
- 注意：透明文字要保证可读（加细微阴影/描边防糊在 K 线上）。

### D2 · 画线工具：浮动可拖动控件 —— ✓LOCKED（待实现可行性最终确认）
- 默认是**一个圆按钮（✎）**，可**手动拖动**到屏幕任意位置。
- **点一下展开**成一行工具条（水平线 / 趋势线 / 删除…，预留更多工具）。
- 展开后**保持展开**；可继续拖动整条；**只有再点 ✎/收起图标才收回**成圆按钮。绝不自动收回。

### D3 · 交易控件：一套，绑定「下单目标周期」 —— ✓LOCKED（方向）
- 全屏**只有一套**买入/卖出/持有，**不每周期各放一套**。user 同意。
- 控件头部显示当前绑定：`下单 · <周期> @ <该周期当前K线收盘价>`。
- **切换下单目标**：点哪个 panel 就绑到哪个（高亮）/ 或控件上「60分|日线」切换钮。
- **取价规则（user 直接拍板，无需再议）**：买/卖价 = **当前绑定周期 K 线「最后一根（当前）K 线的收盘价」**。不存在 bar 内中间点取价——细周期天然给更细分辨率（其最后一根 K 线本身就更细）。
  - ⚠️ 取价/下单的**引擎实现**仍归 **RFC-A**（E3 TradeCalculator / E2 PositionManager）；此规则在此记录，A 照此实现。
- OPEN 子项见 D10：这套控件的**占位形态**（太大问题）。

### D4 · 顶栏指标 —— ✓LOCKED（含宽度与语义）
- 5 格：**总资金 / 持仓成本 / 持仓股数 / 仓位X/5 / 浮动盈亏**（仓位% 与 持仓股数都保留）。
- 每个指标格：**标签在上 / 数值在下**；左右**重新分配宽度**，不要全挤在一起。
- **每格对齐 = 居中对称**：每个指标格内，**标题与数值都相对该格显示区中线居中、左右对称**（不是左对齐）。这样不管数字多少位，都在格中位线对称。
- **总资金**：预留 **8 位数**宽度（`¥99,999,999`，千万级）——起始 10万，留到千万位即可（user 更正：不用留 9 位/亿位）。居中、缩字体不撑宽、不换行。
- **持仓股数**：会随复利+低价股变很大 → 预留 **7 位数**宽度（最高 `9,999,999 股`），居中。✓ user 确认 7 位够（最坏值压测无重叠）。
- **持仓成本/股、浮动盈亏**：量级不夸张（价位级 / 百分比），居中即可。
- **顶栏不能重叠**：之前持仓股数/仓位 UI 重叠 → 重新分配每格宽度 + 居中，最坏值也不挤不叠（v6 用最坏值压测）。
- **持仓成本 = 每股成本（重要语义纠正）**：= 买入价 + 税费(免5/规费等)分摊**到每股**，量级=**股价级**（如 `1,683.50`），**不是**全部持仓的总成本。用途：与现价比，看盈亏关系。
  - ⚠️ 该「每股成本含税费」的**计算口径**牵涉 RFC-A（E3 TradeCalculator 成本基）；B 负责按每股成本显示，A 保证数值口径正确。
- 浮动盈亏 = 现价相对每股成本的百分比（红涨绿跌）。
