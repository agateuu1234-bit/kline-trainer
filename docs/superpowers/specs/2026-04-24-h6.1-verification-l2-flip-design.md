# Hardening-6.1: `verification-before-completion` L2 Per-Skill Flip 设计 Spec

**Status**: drafted 2026-04-24；ζ 渐进路径第 1 步；H6.0 framework (PR #30) + H6.0.1a (PR #31) 已 merged 为前提。

---

## 1. 背景

- H6.0 framework 上线 observe-only 已满一周；L2 `gate_declared_no_invoke` 机制在 `.claude/hooks/skill-invoke-check.sh` 中就绪，per-skill `mode` 字段在 `.claude/config/skill-invoke-enforced.json` 已布设。
- 原"H6.0-flip 统一 flip 全 10 skill"路径证明不可行（大 PR codex review 不收敛）；本 PR 采用 ζ plan §4 的 per-skill microPR 路径（参见 `docs/superpowers/specs/2026-04-22-hardening-6-framework-design.md` §4）。
- 首个翻转目标选 `superpowers:verification-before-completion`，理由：
  - 它是完成声明闸门，误报成本低（挡住"没 invoke 就 claim done"）
  - 合法前序集已完备（subagent-driven-development / executing-plans / test-driven-development / receiving-code-review / systematic-debugging），翻转后正常工作流不会误挡
  - H6.0 observe 期内该 skill 的 drift-log 样本证明 L2 判定按设计工作

## 2. Scope

**In**：
- 翻转 `.claude/config/skill-invoke-enforced.json` 中 `superpowers:verification-before-completion` 的 `mode`：`"observe"` → `"block"`
- 新增单元测试验证翻转后 L2 判定行为
- 更新 spec + implementation plan 文档

**Out（显式不做）**：
- 不改 `.claude/hooks/skill-invoke-check.sh` 本体

- 不改 workflow-rules.json 全局 `enforcement_mode`（仍 `drift-log`）
- 不翻转其他 skill（H6.2-H6.10 各自 microPR）
- 不分层 gate（`l2: "block", l4: "observe"` 等结构性扩展属于 β/γ 方案）

### 3.2 L2 判定行为（翻转前/后对照）

| # | 场景 | last_stage | 声明 gate | 实际 invoke | 翻前（observe） | 翻后（block） |
|---|---|---|---|---|---|---|
| 1 | 正确使用 | `test-driven-development` | `verification-before-completion` | `Skill{verification-before-completion}` | 通过 | 通过 |
| 2 | 只贴 label 不 invoke | 同上 | 同上 | `Read` 等非 Skill 工具 | drift-log、通过 | **block + drift-log** |
| 3 | invoke 错 skill | 同上 | 同上 | `Skill{brainstorming}` | drift-log、通过 | **block + drift-log** |
| 4 | 错 last_stage（L4 同步生效，**非副作用，系设计目标**） | `_initial` | 同上 | `Skill{verification-before-completion}` | drift-log、通过 | **block**（L4 illegal_transition） |

**关于场景 4 的设计意图**：`verification-before-completion` 语义为"即将声称工作完成/通过"。在 `_initial`（session 始点 / push PR 后 reset）时无任何工作可验证，此刻声明 gate 是语义错误。L4 在 `mode: "block"` 下一并生效是框架原设计 — 按 `ζ` 方案每 skill 一个标量 `mode` 同时覆盖 L2 + L4，非 scope 外耦合。

### 3.3 L2 判定流程（既有机制，无改动）

hook `.claude/hooks/skill-invoke-check.sh` 第 498-530 行已有判定：
```
if SKILL_NAME != codex:adversarial-review:
    检查当前 turn tool_uses 中是否存在 name=="Skill" 且 input.skill == SKILL_NAME
    无匹配 且 exempt_rule 未中 →
        drift-log (drift_kind="gate_declared_no_invoke")
        CONFIG_MODE == "block" → block()，退出
        否则 → 通过
```

本 PR 无改动，仅激活判定分支。

## 4. 测试设计

### 4.1 测试类（新增）

文件：`tests/hooks/test_skill_invoke_check.py`

```python
class TestVerificationBeforeCompletionBlockFlip:
    """H6.1: verification-before-completion mode flipped to block.
    复用 _set_mode 显式锁定 "block"（防回滚静默）。"""

    def test_l2_missing_invoke_blocks(self, tmp_path):
        bak = _set_mode("superpowers:verification-before-completion", "block")
        try:
            tp = _write_transcript(
                tmp_path,
                "Skill gate: superpowers:verification-before-completion\n\nx",
                tool_uses=[{"name": "Read", "input": {"file_path": "/a"}}],
            )
            rc, stdout, _ = _run_hook(tp)
            assert '"decision":"block"' in stdout.replace(" ", "")
        finally:
            _restore(bak)

    def test_l2_valid_invoke_passes(self, tmp_path):
        bak = _set_mode("superpowers:verification-before-completion", "block")
        try:
            tp = _write_transcript(
                tmp_path,
                "Skill gate: superpowers:verification-before-completion\n\nx",
                tool_uses=[{"name": "Skill",
                            "input": {"skill": "superpowers:verification-before-completion"}}],
            )
            rc, stdout, _ = _run_hook(tp)
            assert '"decision":"block"' not in stdout.replace(" ", "")
        finally:
            _restore(bak)

    def test_l2_wrong_skill_invoke_blocks(self, tmp_path):
        bak = _set_mode("superpowers:verification-before-completion", "block")
        try:
            tp = _write_transcript(
                tmp_path,
                "Skill gate: superpowers:verification-before-completion\n\nx",
                tool_uses=[{"name": "Skill",
                            "input": {"skill": "superpowers:brainstorming"}}],
            )
            rc, stdout, _ = _run_hook(tp)
            assert '"decision":"block"' in stdout.replace(" ", "")
        finally:
            _restore(bak)
```

### 4.2 测试覆盖矩阵

| L2 情景 | 测试 | 断言 |
|---|---|---|
| 正确 invoke | `test_l2_valid_invoke_passes` | 不 block |
| 缺 invoke | `test_l2_missing_invoke_blocks` | block |
| 错 invoke | `test_l2_wrong_skill_invoke_blocks` | block |

L4 `illegal_transition` 行为不在本 PR 测试范围 — 已有 H6.0 框架测试覆盖（`TestL4LegalTransition.*`）。

## 5. Rollback

Revert 本 PR 的单行 config 改动 → 恢复 `mode: "observe"` → 回到 observe-only。
- 无 schema 变更
- 无 state 迁移
- 无 hook 逻辑改动

## 6. 风险与缓解

| 风险 | 可能性 | 缓解 |
|---|---|---|
| 翻转后某合法工作流被误 block（L2 或 L4） | 低 — observe 期内 drift-log 已采样 | Rollback 成本极低（1 行 revert）；drift-log 保留全部样本供回溯 |
| codex review 提议扩大 scope（多 skill / 改 hook / 加架构层） | 中 — 历史上 H6.0.1d 发生过 | 以本 spec §2 的 "显式 Out" 清单 pushback；超 3 轮 escalate 给用户 |
| 测试 fixture `_set_mode` 破坏其他测试 | 低 — 已有多处调用无 side effect | `_restore` 在 `try/finally` 中执行；与既有用法一致 |
