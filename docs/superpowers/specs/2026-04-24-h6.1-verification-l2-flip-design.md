## 1. 背景

- H6.0 framework 上线 observe-only 已满一周；L2 `gate_declared_no_invoke` 机制在 `.claude/hooks/skill-invoke-check.sh` 中就绪，per-skill `mode` 字段在 `.claude/config/skill-invoke-enforced.json` 已布设。
- 原"H6.0-flip 统一 flip 全 10 skill"路径证明不可行（大 PR codex review 不收敛）；本 PR 采用 ζ plan §4 的 per-skill microPR 路径（参见 `docs/superpowers/specs/2026-04-22-hardening-6-framework-design.md` §4）。
- 首个翻转目标选 `superpowers:verification-before-completion`，理由：
  - 它是"声明了完成验证 gate 但没 invoke"的**声明完整性 gate**（非零 gate 完成声明的 catch-all — 见 §2）
  - 合法前序集 + `_initial` 豁免（D_narrow pattern 1c）覆盖全部正常工作流入口
  - L2 机制在其他 skill 的 drift-log 样本中已验证按设计工作（`systematic-debugging` 7 条 + `brainstorming` 5 条 `gate_declared_no_invoke` 如期触发 drift-log）；本 skill 在 H6.0 observe 期**零样本** — 但 1c 豁免消除了 observe 期最可能误挡的 `_initial` 场景，所以零样本不是安全性障碍

**本 PR 明确不是**：
- 完成声明通用 catch-all — 本 flip 仅在 agent **已声明** `Skill gate: superpowers:verification-before-completion` 时才参与判定。零 gate 完成声明（agent 完全不写 first-line gate 就 claim done）走 L1 global enforcement 路径，当前仍 `drift-log` 模式（受 `workflow-rules.json::skill_gate_policy.enforcement_mode` 控制），不在本 PR scope 内。

### 3.2 L2/L4 判定行为（翻转前/后对照，D_narrow 方案）

| # | 场景 | last_stage | 声明 gate | 实际 invoke | 翻前（observe） | 翻后（block + 1c 豁免）|
|---|---|---|---|---|---|---|
| 1 | 正确使用（合法前序）| `test-driven-development` | `verification-before-completion` | `Skill{verification-before-completion}` | 通过 | 通过 |
| 2 | 只贴 label 不 invoke | 同上 | 同上 | `Read` 等非 Skill 工具 | drift-log、通过 | **L2 block** |
| 3 | invoke 错 skill | 同上 | 同上 | `Skill{brainstorming}` | drift-log、通过 | **L2 block** |
| 4 | **`_initial` + 正确 invoke**（session resume / state reset）| `_initial` | 同上 | `Skill{verification-before-completion}` | drift-log、通过 | **通过**（因 1c 把 verification 加到 `_initial` 的合法后继）|
| 5 | 真非法前序 + 正确 invoke（脑暴后跳验证）| `brainstorming` | 同上 | `Skill{verification-before-completion}` | drift-log、通过 | **L4 block**（保留 feature — brainstorming 不是 `verification-before-completion` 的合法前序）|

**场景 4 设计意图（D_narrow 解法）**：session resume 场景 state 可能不存在（last_stage=_initial），但实际工作可能已完成。加 1c 让 `_initial → verification` 合法，消除 codex 反复担心的"session resume 被 L4 误挡"。

**场景 5 保留意图**：从 `brainstorming` 等真·非法前序跳 verification 仍 L4 block — 这是 ζ 状态机的防"假装验证完成"保护，不该关。

## 4. 测试设计

### 4.1 测试类（新增）

文件：`tests/hooks/test_skill_invoke_check.py`

**设计要点**（codex R1 F1 修正后）：
1. 必须 **pre-seed state file** 设 `last_stage=superpowers:test-driven-development`（legal 前序之一），否则 `_initial` 状态下 L4 会抢先 block，掩盖 L2 检查。
2. 必须通过 `env_extra` 传入 **deterministic `CLAUDE_SESSION_ID`** + `CLAUDE_SESSION_START_UTC`，否则 hook 在 block 模式下 L4 `session_id_absent_l4_fail_closed` 抢先 block（见 `skill-invoke-check.sh:282-285`）。
3. 每个负例 assert **具体 block reason 子串**（不仅 `"decision":"block"`），防止被更早期的 fail-closed 分支"冒充" L2 命中。

```python
class TestVerificationBeforeCompletionBlockFlip:
    """H6.1: verification-before-completion mode flipped to block.
    复用 _set_mode 显式锁定 "block"（防回滚静默）。
    复用 TestL5CodexGate 同款 state seeding + session id 约定。"""

    DETERMINISTIC_SESSION_ID = "01HQTESTAAAAAAAAAAAAAAAAAA"

    def _sid_hash8(self, sid=None):
        import hashlib
        sid = sid or self.DETERMINISTIC_SESSION_ID
        return hashlib.sha256(sid.encode()).hexdigest()[:8]

    def _seed_tdd_stage(self, session_id=None):
        """Pre-seed state file with last_stage=test-driven-development
        so transition → verification-before-completion is legal at L4."""
        import hashlib
        state_dir = Path(".claude/state/skill-stage")
        state_dir.mkdir(parents=True, exist_ok=True)
        wt_h = hashlib.sha256(os.getcwd().encode()).hexdigest()[:8]
        sid_h = self._sid_hash8(session_id)
        sf = state_dir / f"{wt_h}-{sid_h}.json"
        sf.write_text(json.dumps({
            "version": "1",
            "last_stage": "superpowers:test-driven-development",
            "last_stage_time_utc": "2026-04-22T00:00:00Z",
            "worktree_path": os.getcwd(),
            "session_id": session_id or self.DETERMINISTIC_SESSION_ID,
            "drift_count": 0,
            "transition_history": [],
        }))
        return sf

    def _block_env(self):
        return {
            "CLAUDE_SESSION_ID": self.DETERMINISTIC_SESSION_ID,
            "CLAUDE_SESSION_START_UTC": "2026-04-22T00:00:00Z",
        }

    def test_l2_missing_invoke_blocks(self, tmp_path):
        sf = self._seed_tdd_stage()
        bak = _set_mode("superpowers:verification-before-completion", "block")
        try:
            tp = _write_transcript(
                tmp_path,
                "Skill gate: superpowers:verification-before-completion\n\nx",
                tool_uses=[{"name": "Read", "input": {"file_path": "/a"}}],
            )
            rc, stdout, _ = _run_hook(tp, env_extra=self._block_env())
            assert '"decision":"block"' in stdout.replace(" ", "")
            # Assert L2-specific reason (非 session_id_absent / 非 illegal_transition)
            assert "声明但响应未 Skill tool invoke" in stdout
        finally:
            _restore(bak)
            sf.unlink(missing_ok=True)

    def test_l2_valid_invoke_passes(self, tmp_path):
        sf = self._seed_tdd_stage()
        bak = _set_mode("superpowers:verification-before-completion", "block")
        try:
            tp = _write_transcript(
                tmp_path,
                "Skill gate: superpowers:verification-before-completion\n\nx",
                tool_uses=[{"name": "Skill",
                            "input": {"skill": "superpowers:verification-before-completion"}}],
            )
            rc, stdout, _ = _run_hook(tp, env_extra=self._block_env())
            assert '"decision":"block"' not in stdout.replace(" ", "")
        finally:
            _restore(bak)
            sf.unlink(missing_ok=True)

    def test_l2_wrong_skill_invoke_blocks(self, tmp_path):
        sf = self._seed_tdd_stage()
        bak = _set_mode("superpowers:verification-before-completion", "block")
        try:
            tp = _write_transcript(
                tmp_path,
                "Skill gate: superpowers:verification-before-completion\n\nx",
                tool_uses=[{"name": "Skill",
                            "input": {"skill": "superpowers:brainstorming"}}],
            )
            rc, stdout, _ = _run_hook(tp, env_extra=self._block_env())
            assert '"decision":"block"' in stdout.replace(" ", "")
            assert "声明但响应未 Skill tool invoke" in stdout
        finally:
            _restore(bak)
            sf.unlink(missing_ok=True)

    def test_committed_config_is_block(self):
        """Codex R2 F2 fix: guard against _set_mode masking a regressed commit.
        This test reads the ACTUAL file in-repo and asserts committed mode,
        independent of any fixture that forces runtime values."""
        cfg = json.loads(Path(".claude/config/skill-invoke-enforced.json").read_text())
        assert cfg["enforce"]["superpowers:verification-before-completion"]["mode"] == "block", \
            "H6.1 flip regressed: committed config must be 'block', not 'observe'"

    def _seed_stage(self, stage, session_id=None):
        """Generalized version of _seed_tdd_stage for any predecessor."""
        import hashlib
        state_dir = Path(".claude/state/skill-stage")
        state_dir.mkdir(parents=True, exist_ok=True)
        wt_h = hashlib.sha256(os.getcwd().encode()).hexdigest()[:8]
        sid_h = self._sid_hash8(session_id)
        sf = state_dir / f"{wt_h}-{sid_h}.json"
        sf.write_text(json.dumps({
            "version": "1",
            "last_stage": stage,
            "last_stage_time_utc": "2026-04-22T00:00:00Z",
            "worktree_path": os.getcwd(),
            "session_id": session_id or self.DETERMINISTIC_SESSION_ID,
            "drift_count": 0,
            "transition_history": [],

        return sf

    def test_l4_codex_to_verification_still_blocks(self, tmp_path):

        构成 bypass（跳过 writing-plans + TDD）。本测试断言该 transit 仍挡。"""
        sf = self._seed_stage("codex:adversarial-review")

    def test_l2_block_drift_log_matches_monitoring_predicate(self, tmp_path):
        """Codex R9 F2 fix: verify emitted drift-log entry has the exact
        fields (gate_skill / config_mode / drift_kind) the §6 revert
        predicate reads. Without this, post-merge monitoring could silently
        miss real blocks if hook emits malformed entries."""
        sf = self._seed_tdd_stage()
        bak = _set_mode("superpowers:verification-before-completion", "block")
        drift_path = Path(".claude/state/skill-invoke-drift.jsonl")
        drift_path.parent.mkdir(parents=True, exist_ok=True)
        lines_before = drift_path.read_text().count('\n') if drift_path.exists() else 0
        try:
            tp = _write_transcript(
                tmp_path,
                "Skill gate: superpowers:verification-before-completion\n\nx",

            _run_hook(tp, env_extra=self._block_env())

                    e = json.loads(line)
                except json.JSONDecodeError:

            assert relevant, \
                "drift log must include an entry for the blocked gate"
            e = relevant[-1]
            # Revert predicate schema (aligned with §6 row 2):
            assert e["config_mode"] == "block"
            assert e["drift_kind"] in ("gate_declared_no_invoke", "illegal_transition")
        finally:
            _restore(bak)
            sf.unlink(missing_ok=True)

    def test_l4_initial_to_verification_allowed(self, tmp_path):

        sid_h = self._sid_hash8()
        sf = state_dir / f"{wt_h}-{sid_h}.json"
        sf.unlink(missing_ok=True)  # ensure _initial

        bak = _set_mode("superpowers:verification-before-completion", "block")
        try:
            tp = _write_transcript(
                tmp_path,
                "Skill gate: superpowers:verification-before-completion\n\nx",
                tool_uses=[{"name": "Skill",
                            "input": {"skill": "superpowers:verification-before-completion"}}],
            )
            rc, stdout, _ = _run_hook(tp, env_extra=self._block_env())

        finally:
            _restore(bak)
            sf.unlink(missing_ok=True)

    def test_l4_brainstorming_to_verification_still_blocks(self, tmp_path):
```

### 5.2 L4 `_initial` 场景说明（D_narrow 已解决）

原 concern：agent 在 `_initial` 状态下声明 verification gate 会被 L4 挡（R2/R5/R6/R9/R11 F1）。

**D_narrow 方案已在 config 根治**：1c 把 `verification-before-completion` 加进 `_initial` 的 `legal_next_set`，所以 session resume / state reset 场景正确 invoke verification **不再被 L4 block**（行为由 test #7 `test_l4_initial_to_verification_allowed` pin 住）。

仍会 L4 block 的场景（**保留的 feature**）：从 `brainstorming` / `writing-plans` / `codex:adversarial-review` 外的**真·非法前序**直接跳 verification（防"脑暴后假装验证完成"等语义错误），行为由 test #8 pin 住。

如果 L4 block 意外触发：查 `.claude/state/skill-invoke-drift.jsonl` 里 `last_stage_before` 字段判断是不是合理前序、若否 agent 回到正常工作流（per skill-router）即可，不需要任何 override。
